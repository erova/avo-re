## The problem

Most onboarding platforms are optimized for non-technical “overlay editing,” not for building a cohesive product experience.

They tend to be:

<ul className="space-y-2 my-6">
  <li><strong>Expensive</strong> — $1000+/month subscriptions</li>
  <li><strong>Heavy</strong> — 100KB+ bundle sizes</li>
  <li><strong>Inflexible</strong> — clunky visual editors, limited customization</li>
  <li><strong>Vendor lock-in</strong> — no code ownership</li>
</ul>

What I want instead is a small, composable system that behaves like part of the product—not a third-party layer on top of it.

<div className="my-12" />

## The solution

A clean, maintainable onboarding system with:

<ul className="space-y-2 my-6">
  <li><strong>Layered architecture</strong> — Separate logic, config, and UI</li>
  <li><strong>Lightweight</strong> — ~15KB total target</li>
  <li><strong>Full control</strong> — Own the code; customize everything</li>
  <li><strong>Type-safe</strong> — End-to-end TypeScript</li>
  <li><strong>Production-minded</strong> — persistence, events, progress tracking</li>
</ul>

<div className="my-12" />

## UX patterns included

### 1) Interactive tooltips (guided tour)

Guide users through features with positioned tooltips that include:

<ul className="space-y-1 my-4">
  <li>Smart positioning (top, bottom, left, right)</li>
  <li>Arrow + target anchoring</li>
  <li>Optional media (GIFs) for demonstrations</li>
  <li>Focus/attention treatment on target elements</li>
</ul>

<div className="my-8" />

### 2) Progress tracking (checklist)

A floating checklist panel that supports:

<ul className="space-y-1 my-4">
  <li>Completion count (e.g., “2/4 completed”)</li>
  <li>Expandable task list</li>
  <li>Toast notifications on progress</li>
  <li>Visual feedback (checkmarks, strikethrough)</li>
</ul>

<div className="my-8" />

### 3) Help indicators (ambient discovery)

Pulsing “!” badges on interactive elements to show what’s available to explore—without forcing users into a step-by-step flow.

<div className="my-12" />

## Architecture

The system is intentionally split so you can swap UI without rewriting logic.

### Core engine

`onboarding-engine.ts` — a pure TypeScript class with **zero UI dependencies**, responsible for:
- state + persistence
- step/task progression
- event emission (analytics-friendly)
- guardrails (only show once, prerequisites, etc.)

### Configuration

`onboarding-config.ts` — pure data defining flows and steps. This can live in code, load from an API, or be managed in a CMS—without changing UI.

### React integration

`useOnboarding.ts` — a single hook that connects the engine to components. This is the only integration point your app needs.

### UI components

<ul className="space-y-1 my-4">
  <li><strong>OnboardingTooltip</strong> — the guided tour tooltips</li>
  <li><strong>HelpIndicator</strong> — pulsing badges on elements</li>
  <li><strong>ChecklistPanel</strong> — progress tracking UI</li>
</ul>
